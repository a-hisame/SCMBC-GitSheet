テスト@kokmt
github事始め
GitHubのプロジェクトを作成
空のプロジェクトを（ローカルに）クローンgit clone git@github.com:a-hisame/Project.git
何かファイルを作成してコミット
  • git add .
  • git commit -m 'commit comment'
ローカルからGitHubにpush git push origin master
GitHub側にdevelopment branchを作成 git push origin master:development
ローカルにGitHubのブランチをコピー git branch development origin/development
ブランチを移動git checkout development
ファイルを作る。add/commitgit add * / git commit *
コミットをpushgit push origin development
コンフリクト！ もう一度pullgit pull
オートコンフリクト修正に失敗した場合はファイルを直す git add [conflict file] / git commit
コンフリクトを起こしたけどマージを終了したい！ git merge --abort
特定のブランチに移動(現在の変更を破棄する。SVNのrevert git reset --hard origin/development
ブランチをpush（localのbranchをremoteのbranchに設定) git push origin development:master (local:development を origin:masterにpush)
特定のブランチを切る git branch rebase_test development
コンフリクトを起こす場合、元のブランチで更新をかける git checkout development / git pull
移動させたいブランチに移動し、そのブランチを追加したいブランチを指定することでrebaseする。git checkout rebase_test / git rebase development
情報
初期化
リポジトリの作成をするgit init
リポジトリの複製をするgit clone 複製元リポジトリのURL
バージョン管理
変更をコミットするgit commit -a
ひとつ前のコミットをチェックアウトするgit co HEAD^
変更を元に戻すgit reset HEAD --hard
ファイル操作
ファイルをレポジトリの管理対象に追加するgit add ファイル名
ファイルをレポジトリの管理対象に追加を取りやめるgit rm --cached ファイル名
ファイルをローカルレポジトリの最新の状態にするgit co HEAD ファイル名
ファイルを移動するgit mv 古いファイル名 新しいファイル名
ファイルをレポジトリの管理対象から外すgit rm ファイル名
プッシュ操作
ブランチの変更をリモートレポジトリに反映するgit push リモート名 ブランチ名
リモートレポジトリからブランチを削除git push リモート名 :ブランチ名
マージ操作
カレントブランチにローカルブランチをマージするgit merge ブランチ名
カレントブランチにリモートブランチをマージするgit pull リモート名 ブランチ名
git の設定
gitにユーザ名を教えるgit config --add user.name あなたの名前
gitにメールアドレスを教えるgit config --add user.email あなたのメールアドレス
エイリアスを設定するgit config --add alias.エイリアス名 コマンド名
